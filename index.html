    🏢 Company: HPEL

    🛠 Tech Stack: Java, Spring Boot, MongoDB, React, JWT, Role-Based Auth, REST APIs

    🔍 Project Overview

    Designed to manage and track coolers provided by distributors to retailers, ensuring proper allocation, usage, and asset control. The system includes a request approval workflow that tracks installation requests from Sales to final confirmation, with process visibility across multiple departments.

    🔧 Key Features

    • Multi-Stage Workflow: 10-step approval flow covering Sales, Finance, Logistics, Admin.
    • Role-Based Access: Auth for 8 roles using JWT + Spring Security
    • Nested MongoDB Models: One parent request with nested department status
    • File Upload & Storage: Upload agreements/receipts per stage
    • Smart Status Logic: Skips finance for replacements; routes properly on rejection
    • Google Form Integration: Sales team initiates request via embedded form
